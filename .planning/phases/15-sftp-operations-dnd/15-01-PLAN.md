# 15-01-PLAN: Backend Progress Infrastructure

## Goal
Establish the foundation for tracking file transfers and reporting their progress via Server-Sent Events (SSE).

## Tasks

<task>
<action>
Implement `TransferManager` in `be/internal/ssh/transfer_manager.go`.
Define `TransferStatus` struct (BytesTransferred, TotalBytes, Status, Error) and `TransferManager` type with a thread-safe map to store these statuses by UUID.
Include methods: `CreateTransfer`, `UpdateProgress`, `SetComplete`, `SetError`, `GetStatus`.
</action>

</files>
<verify>
Unit test for `TransferManager` ensuring thread-safe updates and retrieval.
</verify>
<done>
`TransferManager` implemented with thread-safe storage and passed unit tests for status lifecycle.
</done>
</task>

<task>
<action>
Implement `ProgressWriter` in `be/internal/ssh/progress.go`.
Create a struct that implements `io.Writer` and takes a `transferId` and `TransferManager` to update progress on every `Write` call.
</action>

<task>
<action>
Update `SFTPHandler` in `be/internal/api/sftp.go` to use `TransferManager`.
1. Add `TransferManager` to `SFTPHandler` struct and initialize it in `server/main.go`.
2. Update `Transfer` handler:
   - Generate `transferId`.
   - Start goroutine:
     - `Stat` source file to get `TotalBytes`.
     - `CreateTransfer` in manager.
     - Wrap staging and writing with `ProgressWriter`.
     - `SetComplete` or `SetError` when done.
   - Return `transferId` immediately.
3. Update `Upload` handler similarly.
4. Add `CheckExists` endpoint or logic to support D-12.
</action>

</task>
