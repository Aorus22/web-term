# Phase 20 Plan 1: Clipboard View

## Overview
Implementation of a real-time clipboard monitoring feature for SSH target machines (Windows and Linux Desktop).

## Objective
Create a lightweight helper program to listen to clipboard events on target machines, deploy it via SSH/SFTP, and display a searchable history in the UI.

## Tasks

### Task 1: Helper Program (be/cmd/clip-helper/)

- [ ] Implement clipboard listener using `golang.design/x/clipboard`
- [ ] Implement Base64 encoding for `stdout` communication
- [ ] Add build script/command to cross-compile for Windows (amd64) and Linux (amd64)

### Task 2: Backend Integration
- [ ] Add `be/internal/ssh/clipboard.go` to manage helper deployment and execution
- [ ] Implement OS detection for target SSH connections

### Task 3: Frontend Implementation

## Technical Details

### Helper Protocol
- The helper prints to `stdout` whenever a clipboard change occurs
- Format: `CLIP:[base64_content]\n`
- This allows the backend to easily parse lines and ignore other potential output

### Deployment Strategy
- Backend checks if the helper exists on the target
- If not, or if version mismatch, it uploads the binary
- On Linux: `chmod +x` is required after upload
- Execution: `nohup ./web-term-clip > /dev/null 2>&1 &`
