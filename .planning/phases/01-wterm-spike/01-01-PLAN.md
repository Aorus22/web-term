```json
// Client → Server (JSON text frame — connection request)
{"type":"connect","host":"1.2.3.4","port":22,"user":"root","password":"secret"}

// Client → Server (JSON text frame — resize)
{"type":"resize","cols":120,"rows":40}

// Client → Server (binary frame — user keystrokes)
raw bytes

// Server → Client (binary frame — SSH output)
raw bytes
```

<task type="auto">
  <name>Task 1: Go WebSocket SSH Proxy Backend</name>
  <files>be/go.mod, be/main.go</files>
  <action>
1. Initialize Go module in `be/`:
   ```
   cd be && go mod init webterm
   go get github.com/gorilla/websocket
   go get golang.org/x/crypto/ssh
   ```

2. Create `be/main.go` — a single-file WebSocket SSH proxy server:
   - Listen on `127.0.0.1:8080`
   - Serve the WebSocket endpoint at `/ws`
   - Use `gorilla/websocket.Upgrader` with `CheckOrigin: return true` (spike, localhost only)
   - On WebSocket connect, wait for a JSON text frame with connection params:
     ```json
     {"type":"connect","host":"...","port":22,"user":"...","password":"..."}
     ```
   - SSH dial using `golang.org/x/crypto/ssh`:
     - `ssh.ClientConfig` with `ssh.Password(password)` auth
     - `HostKeyCallback: ssh.InsecureIgnoreHostKey()` (spike only)
   - Request PTY: `session.RequestPty("xterm-256color", rows, cols, ssh.TerminalModes{})`
   - Start shell: `session.Shell()`
   - Bidirectional pipe using two goroutines:
     - Goroutine 1: `io.Copy(sshStdin, wsReader)` — WebSocket binary messages → SSH stdin
     - Goroutine 2: `io.Copy(wsWriter, sshStdout)` — SSH stdout → WebSocket binary messages
   - Handle JSON text frames for resize: parse `{"type":"resize","cols":N,"rows":N}`, call `session.WindowChange(rows, cols)` via `ssh.Session.SendRequest("window-change", false, ...)`
   - Handle disconnect: close SSH session and WebSocket on either side disconnect
   - Add simple CORS headers for dev: `Access-Control-Allow-Origin: *`
   - Graceful shutdown on SIGINT

3. Key implementation details:
   - Use `websocket.BinaryMessage` for SSH data, `websocket.TextMessage` for JSON control
   - Use `conn.NextReader()` to distinguish message types
   - Set reasonable buffer sizes: ReadBufferSize: 4096, WriteBufferSize: 4096
   - Use context with cancel for goroutine cleanup
   - Log connection events to stdout (connect, disconnect, errors)
  </action>

    <automated>cd be && go build -o webterm . && echo "BUILD_OK"</automated>
  </verify>
  <done>Go binary builds successfully. WebSocket endpoint at /ws accepts connections, dials SSH, and pipes bidirectional I/O with resize support.</done>

  <files>fe/package.json, fe/vite.config.ts, fe/tsconfig.json, fe/tsconfig.app.json, fe/index.html, fe/src/main.tsx, fe/src/App.tsx, fe/src/App.css</files>

   ```
   cd fe && npm create vite@latest . -- --template react-ts
   npm install
   npm install @wterm/dom @wterm/react
   ```

2. Configure `fe/vite.config.ts`:
   - Set server port to 5173 (default)
   - Add proxy for `/ws` → `ws://127.0.0.1:8080` for development:
     ```ts
     server: {
       proxy: {
         '/ws': {
           target: 'ws://127.0.0.1:8080',
           ws: true,
         }
       }
     }
     ```

3. Create `fe/src/App.tsx` — main terminal app:
   - Import `Terminal` and `useTerminal` from `@wterm/react`
   - Import `@wterm/react/css` for terminal styles
   - State: `connected: boolean`, `ws: WebSocket | null`
   - Connection form (shown when not connected):
     - Input fields: Host, Port (default 22), Username, Password
     - "Connect" button
     - Minimal styling — dark background, light text, centered form
   - On connect:
     - Create WebSocket to `ws://127.0.0.1:5173/ws` (proxied to Go backend)
     - On ws.onopen: send JSON `{"type":"connect","host":...,"port":...,"user":...,"password":...}`
     - On ws.onmessage (binary): call `write(event.data)` to terminal
     - Terminal `onData`: send data as binary WebSocket frame via `socket.send(new TextEncoder().encode(data))`
     - Terminal `onResize(cols, rows)`: send JSON `{"type":"resize","cols":cols,"rows":rows}`
   - Terminal component props:
     ```tsx
     <Terminal
       ref={ref}
       autoResize
       theme="solarized-dark"
       cursorBlink
       onData={handleData}
       onResize={handleResize}
     />
     ```
   - Show terminal when connected, show form when disconnected
   - Handle ws.onclose: set connected=false, show form again
   - Handle ws.onerror: log error, show connection error message

4. Create `fe/src/App.css`:
   - Full viewport terminal: `html, body, #root { height: 100%; margin: 0; }`
   - Terminal container fills available space
   - Connection form: centered card with dark background, basic Vercel-inspired minimalism
   - Dark theme: `#000` or `#0a0a0a` background for the form area

5. Keep `fe/src/main.tsx` simple — just render `<App />`
