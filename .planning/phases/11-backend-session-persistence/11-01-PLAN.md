<task type="auto">
  <name>Task 1: Implement SessionManager and RingBuffer</name>
  <files>be/internal/ssh/session_manager.go</files>
  <action>
    Create a thread-safe `SessionManager` in `be/internal/ssh/session_manager.go`.
    - Define `ManagedSession` struct:
      - `ID` (string, UUID)
      - `SSHClient` (*ssh.Client)
      - `SSHSession` (*ssh.Session)
      - `Stdin` (io.WriteCloser)
      - `Host`, `User`, `Port`, `ConnectionID` (metadata)
      - `Buffer` (fixed-size byte slice/ring buffer for scrollback, e.g., 64KB)
      - `WS` (*websocket.Conn) - current active WebSocket
      - `LastSeen` (time.Time) - for cleanup
      - `mu` (sync.Mutex)
    - Implement `NewSessionManager()`, `CreateSession()`, `GetSession()`, `RemoveSession()`.
    - Implement a background goroutine `CleanupWorker` that runs every minute and removes sessions with `WS == nil` and `LastSeen` older than 10 minutes.
    - Implement a simple `RingBuffer` or use a byte slice with offset for storing the last N bytes of terminal output.
  </action>
  <verify>
    <automated>go test ./internal/ssh/...</automated>
  </verify>
  <done>SessionManager exists and can store/retrieve/cleanup sessions with metadata.</done>
</task>

<task type="auto">
  <name>Task 2: Refactor WebSocket Proxy for Persistence</name>
  <files>be/internal/ssh/proxy.go, be/internal/ssh/types.go</files>
  <action>
    Refactor `HandleWebSocket` in `be/internal/ssh/proxy.go` to use the global `SessionManager`.
    - Update `ConnectMessage` in `types.go` to add `Type` ("connect" | "attach") and `SessionID`.
    - In `HandleWebSocket`:
      - If `Type == "connect"`:
        - Proceed with existing SSH dial logic.
        - Create a `ManagedSession` in `SessionManager`.
        - Start a goroutine that reads from `SSHSession` stdout/stderr and writes to BOTH the `ManagedSession.Buffer` and the current `WebSocket`.
      - If `Type == "attach"`:
        - Look up session in `SessionManager` by `SessionID`.
        - If found, bind current WebSocket to the session.
        - **CRITICAL**: Send the current `Buffer` (scrollback) to the client as binary messages immediately.
        - Update `LastSeen` and resume forwarding.
      - On WebSocket close:
        - Set `ManagedSession.WS = nil` and `LastSeen = time.Now()`.
        - DO NOT close the SSH session (unless explicitly requested or cleanup timeout hits).
    - Implement an explicit "disconnect" message handling to allow users to close sessions permanently.
  </action>
  <verify>
    <automated>go build ./cmd/server/...</automated>
  </verify>
  <done>WebSocket handler supports re-attaching to existing sessions and preserves output in a buffer.</done>
</task>
