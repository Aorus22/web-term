<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend -> SFTP API | Untrusted input (connectionId, path) crosses here |
| Backend -> Remote Host | Credentials (password, keys) leave the backend via SSH |
| Backend -> Local FS | API triggered filesystem operations on the host machine |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Information Disclosure | `GET /api/sftp/ls` | mitigate | Ensure path normalization and prevent traversal outside of OS permissions (D-04). |
| T-13-02 | Elevation of Privilege | Local FS access | mitigate | Operations run with the same privileges as the WebTerm backend process. Advise users in docs. |
| T-13-03 | Tampering | `POST /api/sftp/upload` | mitigate | Use streaming to avoid memory exhaustion (D-06, D-08). |
| T-13-04 | Information Disclosure | SSH Credentials | mitigate | Decrypt credentials only at the moment of connection (D-03). |
</threat_model>
