  <action>
    Modify `be/internal/ssh/proxy.go` to add two features:

    **Feature A: cwd injection on connect (for duplicated sessions)**

    After `session.Shell()` succeeds (line ~225) and BEFORE sending the "connected" JSON message:

    ```go
    // If a working directory is specified, cd into it after shell starts
    if connectMsg.Cwd != "" {
        cdCmd := fmt.Sprintf(" cd '%s'\n", connectMsg.Cwd)
        stdinPipe.Write([]byte(cdCmd))
    }
    ```

    Use single quotes around the path and a space prefix before `cd` to prevent it from appearing in shell history. Write this AFTER `Shell()` but BEFORE the "connected" message so the cd executes before the user sees the prompt.

    **Feature B: get-cwd message handler**

    This is the more complex part. The challenge: we need to get the cwd of a RUNNING shell session by injecting a command into stdin and parsing the stdout response, without disrupting the user's terminal output.

    Implementation approach using a stdout interception buffer:

    1. Add a `cwdPending` bool and `cwdBuffer` bytes.Buffer to the handler's local scope (before the forwarding goroutines).

    2. In the WebSocket read goroutine (the one that currently only handles "resize" messages), add handling for a "get-cwd" text message:

    ```go
    } else if ctrl.Type == "get-cwd" {
        // Inject pwd command with unique markers into the running shell
        cwdPending = true
        cwdBuffer.Reset()
        // Use a marker that won't appear in normal terminal output
        marker := "__WTERM_CWD_END__"
        cmd := fmt.Sprintf(" pwd && echo %s\n", marker)
        stdinPipe.Write([]byte(cmd))
    }
    ```

    3. Modify the `forward` function (or the stdout forwarding goroutine) to intercept output when `cwdPending` is true. Instead of a separate function, modify the `go forward(stdoutPipe)` call to be an inline goroutine:

    Replace `go forward(stdoutPipe)` with:
    ```go
    go func() {
        buf := make([]byte, 4096)
        for {
            select {
            case <-ctx.Done():
                return
            default:
            }
            n, err := stdoutPipe.Read(buf)
            if n > 0 {
                data := buf[:n]
                if cwdPending {
                    cwdBuffer.Write(data)
                    // Check if we've received the end marker
                    if bytes.Contains(cwdBuffer.Bytes(), []byte("__WTERM_CWD_END__")) {
                        cwdPending = false
                        // Extract the path: it's the line between the marker start and "__WTERM_CWD_END__"
                        content := cwdBuffer.String()
                        // The pwd output is followed by the marker. Parse it.
                        lines := strings.Split(content, "\n")
                        var path string
                        markerFound := false
                        for _, line := range lines {
                            trimmed := strings.TrimSpace(line)
                            if markerFound {
                                break
                            }
                            if strings.Contains(trimmed, "__WTERM_CWD_END__") {
                                // This line has the marker. The path is before it or on the previous line.
                                beforeMarker := strings.Split(trimmed, "__WTERM_CWD_END__")[0]
                                if beforeMarker != "" {
                                    path = beforeMarker
                                }
                                markerFound = true
                            } else if trimmed != "" {
                                path = trimmed
                            }
                        }
                        // Send the cwd response
                        cwdResp, _ := json.Marshal(CwdResponseMessage{
                            Type: "cwd",
                            Path: path,
                        })
                        wsWrite(websocket.TextMessage, cwdResp)
                        // Send ALL buffered output to the client (includes the pwd output, which is fine)
                        wsWrite(websocket.BinaryMessage, cwdBuffer.Bytes())
                        cwdBuffer.Reset()
                        continue
                    }
                    // Still waiting for marker, don't forward yet (buffer it)
                    continue
                }
                if err := wsWrite(websocket.BinaryMessage, data); err != nil {
                    cancel()
                    return
                }
            }
            if err != nil {
                cancel()
                return
            }
        }
    }()
    ```

    4. Add the required imports at the top: `"bytes"` and `"strings"` (if not already there).

    5. Keep the `go forward(stderrPipe)` as-is (stderr doesn't need interception).

    **Important notes:**
    - The `bytes` package import must be added to the import block
    - The `strings` package must be added if not already there
    - The cwd interception buffers output until the marker is found, then flushes everything — the user sees a brief pause then the pwd output appears in their terminal (this is acceptable UX since "Duplicate" is an explicit action)
    - The `cwdBuffer` variable needs to be declared in the handler scope: `var cwdPending bool; var cwdBuffer bytes.Buffer`
    - The `cwdBuffer` and `cwdPending` are accessed from two goroutines (read goroutine sets cwdPending, stdout goroutine reads it). Use simple bool flag since Go's memory model guarantees single-word reads are atomic. Or use a small mutex if preferred.
