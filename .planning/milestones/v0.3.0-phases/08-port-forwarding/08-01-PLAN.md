<task type="auto">
  <name>Task 2: Tunnel manager — SSH dial, net.Listen, goroutine lifecycle</name>
  <files>be/internal/ssh/tunnel.go</files>
  <action>
Create `be/internal/ssh/tunnel.go` with a `TunnelManager` struct that manages active port forwarding tunnels in-memory (per D-01, D-04, D-06, D-10, D-11).

**TunnelManager struct:**
```go
type TunnelManager struct {
    mu       sync.Mutex
    tunnels  map[string]*ActiveTunnel  // keyed by forward ID
    db       *gorm.DB
    cfg      *config.Config
}

type ActiveTunnel struct {
    Listener   net.Listener
    SSHClient  *ssh.Client
    Cancel     context.CancelFunc
    Error      string
}
```

**NewTunnelManager(db *gorm.DB, cfg *config.Config) *TunnelManager** — constructor.

**Start(forwardID string, conn db.Connection, localPort int, remotePort int) error:**
1. Lock mutex, check if forward is already active — if so, return error.
2. Attempt `net.Listen("tcp", fmt.Sprintf("127.0.0.1:%d", localPort))` — per D-10. If this fails (EADDRINUSE), return a descriptive error like `"Port {localPort} is already in use"`. This is the port conflict detection.
3. Dial SSH to `conn.Host:conn.Port` using the SAME auth logic from `proxy.go` (per D-04):
   - If `conn.AuthMethod == "key"` and `conn.SSHKeyID != nil`: load SSHKey from DB, decrypt with `config.DecryptWithAAD`, parse private key, create signer, use `ssh.PublicKeys(keySigner)`.
   - If `conn.AuthMethod == "password"`: decrypt `conn.Encrypted` with `config.Decrypt`, use `ssh.Password` + `ssh.KeyboardInteractive` (same dual-auth pattern as proxy.go).
   - Use `ssh.InsecureIgnoreHostKey()` (same as existing proxy.go).
4. Store the `net.Listener` and `*ssh.Client` in the `tunnels` map under `forwardID`.
5. Launch a background goroutine that:
   - Accepts connections on the listener in a loop
   - For each accepted connection: `sshClient.Dial("tcp", fmt.Sprintf("localhost:%d", remotePort))` (per D-06, remote host is always localhost relative to SSH server)
   - `io.Copy` in both directions between the local connection and the SSH-tunneled connection, using two goroutines per accepted connection
   - Use `context.WithCancel` for clean shutdown
   - On error, store error string in `ActiveTunnel.Error`
6. If any step fails after listener bind, close the listener and SSH client before returning error.

**Stop(forwardID string) error:**
1. Lock mutex, get tunnel from map.
2. Call `Cancel()` to stop the accept loop.
3. Close the `net.Listener`.
4. Close the `ssh.Client`.
5. Remove from map.

**IsActive(forwardID string) bool** — returns whether tunnel exists in map.

**GetError(forwardID string) string** — returns stored error string, empty if no error.

**StopAll()** — stops all active tunnels (for graceful shutdown).

Key design points:
- Per D-04: Each forward has its own SSH connection, completely decoupled from terminal WebSocket sessions.
- Per D-06: Standard Go SSH forwarding — `net.Listener` accepts, each accepted conn gets its own `sshClient.Dial` + bidirectional `io.Copy`.
- Per D-11: Forwards are independent — closing a terminal tab does NOT affect tunnels.
- Per D-10: Port conflict is caught at `net.Listen` time — no need for a separate port registry.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → REST API (`/api/forwards`) | Untrusted input — user creates/starts forwards via HTTP |
| Go backend → SSH server | Outbound SSH connection using stored credentials |
| Local network → localhost listener | Exposed port on 127.0.0.1 accepts connections from local processes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | CreateForward | mitigate | Validate all input fields: name non-empty, connection_id exists in DB, ports in 1-65535 range |
| T-08-02 | Elevation | StartForward | mitigate | Bind to 127.0.0.1 only (not 0.0.0.0) — prevents external access to forwarded ports |
| T-08-03 | Spoofing | SSH dial | accept | Uses InsecureIgnoreHostKey same as existing proxy.go — project-wide constraint, not this phase |
| T-08-04 | Information | Decrypt creds | mitigate | Decrypt connection credentials only at Start time, never store in PortForward model |
| T-08-05 | DoS | CreateForward | accept | No auth system (project-wide constraint). Forward count limit could be added later |
