## Color

### Light Theme (`:root` — already defined in index.css)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(1 0 0)` (background) | Main content area, terminal background |
| Secondary (30%) | `oklch(0.97 0 0)` (muted) / `oklch(0.985 0 0)` (sidebar) | Sidebar, header, cards, inactive tabs |
| Accent (10%) | `oklch(0.205 0 0)` (primary) | Active tab indicator, theme toggle, keyboard shortcut hints |
| Destructive | `oklch(0.577 0.245 27.325)` | Error state icons, connection error indicators |

### Dark Theme (`.dark` — already defined in index.css)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(0.145 0 0)` (background) | Main content area, terminal background |
| Secondary (30%) | `oklch(0.269 0 0)` (muted/secondary) | Sidebar, header, cards, inactive tabs |
| Accent (10%) | `oklch(0.922 0 0)` (primary) | Active tab indicator, theme toggle, keyboard shortcut hints |
| Destructive | `oklch(0.704 0.191 22.216)` | Error state icons, connection error indicators |

Accent reserved for: active tab underline/bg, theme toggle button, keyboard shortcut badge text, primary CTA buttons.

### Connection Status Colors (NEW — per TAB-04)

| Status | Color Token | Visual |
|--------|------------|--------|
| connected | `text-green-500` (oklch green, no custom token needed) | Solid green dot, 6px, on tab |
| connecting | `text-yellow-500` (oklch yellow) | Pulsing yellow dot, 6px, CSS `animate-pulse` |
| disconnected | `text-muted-foreground` | Empty ring (border-only circle), 6px, muted color |
| error | `text-destructive` | Red dot, 6px, using destructive token |

**Implementation:** Use lucide-react `Circle` icon at `h-1.5 w-1.5` (6px) with color classes, placed before the tab label text.

### Theme Toggle (NEW — per UI-02)

**Mechanism:**
1. On app load, detect system preference via `window.matchMedia('(prefers-color-scheme: dark)')`
2. Store user preference in `localStorage` key `webterm-theme` with values: `'light'` | `'dark'` | `'system'`
3. Toggle adds/removes `.dark` class on `<html>` element (matches existing `@custom-variant dark (&:is(.dark *))` in index.css)
4. Sync terminal theme: when theme changes, update @wterm/react terminal colors via its theme prop (foreground/background CSS variables)

**Toggle button placement:** Right side of the header bar, after the TabBar area. Icon: `Sun` (lucide) for light mode / `Moon` for dark mode / `Monitor` for system. Use `dropdown-menu` for 3 options (Light / Dark / System).

**Terminal theme sync:**
- Light: terminal background `oklch(1 0 0)`, foreground `oklch(0.145 0 0)`
- Dark: terminal background `oklch(0.145 0 0)`, foreground `oklch(0.985 0 0)`
- Apply via @wterm/react theme configuration object, reading from CSS variables at runtime

## Component Inventory

### Enhance: TabBar.tsx

**Current state:** Renders tabs with label + close (×) button, click to switch, click × to close.

**New additions:**

1. **Connection status dot** (TAB-04): 6px colored dot before the tab label
   - Connected: solid green `Circle` icon
   - Connecting: pulsing yellow `Circle` icon with `animate-pulse`
   - Disconnected: muted `Circle` icon with `fill-none` and `stroke-current`
   - Error: destructive `Circle` icon

2. **Tab close confirmation** (TAB-03): When closing a tab that is `connected`, show a brief confirmation tooltip:
   - Use shadcn `tooltip` with `TooltipContent` showing "Disconnect from {host}?"
   - Only show for connected sessions; disconnecting/disconnected tabs close immediately
   - Clicking × on connected tab → shows tooltip with "Disconnect" and "Cancel" buttons
   - This ensures no zombie SSH processes (TAB-03)

3. **Tab overflow scroll** (implicit from TAB-01): Already handled by `overflow-x-auto no-scrollbar` class

4. **Keyboard shortcut hints**: On tab hover, show tooltip with "Ctrl+W to close" using shadcn `tooltip`

### New: ThemeToggle component

**Location:** `fe/src/components/ThemeToggle.tsx`

**Design:**
- Button with icon (Sun/Moon/Monitor based on current theme)
- Opens a `dropdown-menu` with 3 options:
  - ☀️ Light — description: "Light theme"
  - 🌙 Dark — description: "Dark theme"
  - 💻 System — description: "Follow system preference"
- Active option has a checkmark (using `dropdown-menu` `CheckboxItem` pattern)
- Use existing shadcn `dropdown-menu` and `button` components

**Size:** `h-8 w-8` icon button, `variant="ghost"` — matches sidebar toggle button pattern

### New: useKeyboardShortcuts hook

**Location:** `fe/src/hooks/use-keyboard-shortcuts.ts`

**Behavior:**
- `Ctrl+T`: Opens quick-connect or creates a new empty tab (trigger the quick-connect flow)
- `Ctrl+W`: Close the active tab (same as clicking ×)
- `Ctrl+Tab` / `Ctrl+Shift+Tab`: Cycle through open tabs forward/backward

**Implementation notes:**
- Use `useEffect` with `keydown` event listener on `window`
- Prevent default browser behavior for these combinations (`e.preventDefault()`)
- Only active when terminal is not in a state that captures these keys (terminal captures Ctrl+W for delete-word in bash — need to check if terminal has focus and skip the shortcut, OR use a different approach)
- **Critical UX decision:** When the terminal has focus, Ctrl+W and Ctrl+T should pass through to the terminal (they are standard shell keybindings). Only intercept when focus is outside the terminal (sidebar, header, empty state).
- Alternative: Use `Alt+T`, `Alt+W`, `Alt+Tab` instead to avoid terminal conflicts — but requirements specify Ctrl. **Resolution:** Intercept Ctrl+T/Ctrl+W/Ctrl+Tab only when `document.activeElement` is NOT inside a `.wterm` terminal element. Show a small toast/banner on first use explaining that shortcuts work when terminal is not focused.

### New: useTheme hook

**Location:** `fe/src/hooks/use-theme.ts`

**Manages:**
- System preference detection (`matchMedia`)
- User preference persistence (`localStorage`)
- `.dark` class toggling on `<html>`
- Theme state: `'light'` | `'dark'` | `'system'`
- Resolved theme (actual applied): `'light'` | `'dark'`

## Interaction Specifications

### Tab Interactions

| Action | Trigger | Response |
|--------|---------|----------|
| Switch tab | Click tab / Ctrl+Tab | Active tab highlighted, terminal pane visibility toggled |
| Close tab (disconnected) | Click × / Ctrl+W | Tab removed, SSH session cleaned up |
| Close tab (connected) | Click × / Ctrl+W | Show disconnect confirmation tooltip → on confirm: close tab, disconnect SSH |
| New tab | Ctrl+T | Focus quick-connect input in sidebar |
| Tab hover | Hover over tab | Show close button (opacity-0 → opacity-100), show keyboard shortcut tooltip |

### Theme Toggle Interactions

| Action | Trigger | Response |
|--------|---------|----------|
| Toggle theme | Click icon → select option | UI switches theme, terminal syncs, preference saved |
| System preference changes | OS setting change | If user selected "System", auto-switch theme |

### Keyboard Shortcut Guard

| Shortcut | When Active | When Terminal Focused |
|----------|------------|----------------------|
| Ctrl+T | Opens new connection | Passes through to terminal |
| Ctrl+W | Closes active tab | Passes through to terminal (shell delete-word) |
| Ctrl+Tab | Cycles tabs | Passes through to terminal |

**Detection:** Check `document.activeElement` — if it's inside an element with `.wterm` class (or the terminal container), skip the shortcut handler.

## Layout Specification

### Header Bar (enhanced)

```
┌──────────────────────────────────────────────────────────────────┐
│ [≡] │ [tab1 ●user@host] [tab2 ○user@host] [tab3 ◎user@host]... │ [🌙]│
│     │                                                           │     │
└──────────────────────────────────────────────────────────────────┘
  ↑              ↑                                                 ↑
  sidebar     tab bar (flex-1, overflow-x-auto)              theme toggle
  toggle      ● = connected (green)                              (h-8 w-8)
              ○ = disconnected (muted)                            ghost btn
              ◎ = connecting (yellow, pulse)
```

- Header height: `h-12` (48px, unchanged)
- Theme toggle: right-aligned, `ml-auto` pushes it to the far right
- Tab bar: fills remaining space between sidebar toggle and theme toggle

### Tab Design Detail

```
 ┌──────────────────────┐
 │ ● username@host   ×  │
 └──────────────────────┘
   ↑                  ↑
   status dot (6px)   close btn (3px icon, opacity-0 → hover:opacity-100)
```

- Active tab: `bg-background border-border text-foreground` (unchanged)
- Inactive tab: `bg-transparent border-transparent text-muted-foreground hover:text-foreground hover:bg-muted/50` (unchanged)
- Active tab gets a 2px bottom accent line using `border-b-2 border-primary` or a `::after` pseudo-element
- Tab min-width: 120px (`min-w-[120px]`)
- Tab max-width: 200px (`max-w-[200px]`)
- Label truncation: `truncate` class (already in use)
