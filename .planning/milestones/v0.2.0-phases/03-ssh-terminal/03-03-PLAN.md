**Part A: Create supporting components**

1. Create `fe/src/components/TabBar.tsx`:
   - Reads `sessions` and `activeSessionId` from app store
   - Renders a horizontal tab bar in the header area
   - Each tab shows: connection label (or `user@host` for quick-connect) with a small × close button on the right
   - Active tab has distinct style: `bg-background border-b-2 border-primary` (matches existing tab placeholder style)
   - Clicking a tab calls `setActiveSession(id)`
   - Clicking × calls `removeSession(id)` which triggers disconnect in TerminalPane cleanup
   - If no sessions: render nothing (null) — the existing "No active sessions" placeholder shows in the content area
   - Use Lucide `X` icon for close button

2. Create `fe/src/features/terminal/PasswordPrompt.tsx`:
   - Props: `{ host: string; username: string; onConnect: (password: string) => void; onCancel: () => void }`
   - Centered in terminal pane area (same position as "No active sessions" placeholder)
   - Shows: "Connect to {username}@{host}" heading, password input field (type="password"), Connect button, Cancel button
   - Enter key in password field triggers connect
   - Per D-02/D-03: inline prompt, NOT modal or sheet
   - Use shadcn/ui Input and Button components

3. Create `fe/src/features/terminal/ReconnectOverlay.tsx`:
   - Props: `{ host: string; onReconnect: () => void; onDismiss: () => void }`
   - Semi-transparent overlay on top of the terminal pane (absolute positioned)
   - Shows: "Connection to {host} lost" message + Reconnect button + Close button
   - Per UI-04: clear reconnection prompt on disconnect
   - Use shadcn/ui Button component

**Part B: Wire connections to terminal sessions**

4. Modify `fe/src/features/connections/components/ConnectionList.tsx`:
   - When user clicks a connection:
     a. Generate a new session ID: `crypto.randomUUID()`
     b. Check if connection has a stored password (connection.password from API response):
        - **Has password (D-01):** Call `addSession({...SSHSession, status: 'connecting'})` then `setActiveSession(id)`. The TerminalPane auto-connects with `initialConnect={{ connectionId: connection.id }}`.
        - **No password (D-02):** Call `addSession({...SSHSession, status: 'connecting'})` then `setActiveSession(id)`. The TerminalPane renders PasswordPrompt instead of immediately connecting.
     c. If a session already exists for this connection ID, switch to it instead of creating a new one: `setActiveSession(existingSession.id)`

   Import `useAppStore` for session management. Import `useConnections` to get connection data (already imported).

5. Modify `fe/src/features/connections/components/QuickConnect.tsx`:
   - When user submits quick-connect (Enter with `user@host[:port]` format):
     a. Parse the input: split by `@` to get username and hostPort, split hostPort by `:` for host and optional port (default 22)
     b. Generate session ID
     c. Call `addSession({id, host, port, username, label: \`${user}@${host}\`, status: 'connecting', isQuickConnect: true})`
     d. `setActiveSession(id)` — TerminalPane shows PasswordPrompt (D-03)
   - Keep existing behavior: selecting a saved connection from dropdown uses ConnectionList flow

**Part C: Update App.tsx**

6. Modify `fe/src/App.tsx`:
   - Replace the existing tab bar placeholder `{/* Tabs will go here in Phase 3 */}` with `<TabBar />`
   - Replace the "No active sessions" placeholder content area with:
     ```tsx
     {activeSessionId ? (
       <TerminalPane key={activeSessionId} sessionId={activeSessionId} />
     ) : (
       // Existing "No active sessions" placeholder
     )}
     ```
   - Use `key={activeSessionId}` on TerminalPane so React remounts when switching sessions (each session has its own WebSocket/terminal)
   - Import TabBar and TerminalPane

   **Important:** The `key={activeSessionId}` pattern means each session gets its own TerminalPane instance. When switching tabs, the previous terminal unmounts (WebSocket disconnects). This is the simple v1 approach — keeping multiple sessions alive simultaneously is Phase 4. For now, switching tabs disconnects the previous session.

   Actually, per CONTEXT.md "Tab bar showing active sessions (basic — no multi-tab management, that's Phase 4)" — we should keep sessions alive when switching tabs. Instead of key-based remounting, render ALL session TerminalPanes but show only the active one:

   ```tsx
   <div className="flex-1 relative">
     {sessions.map((session) => (
       <div
         key={session.id}
         className={cn("absolute inset-0", session.id !== activeSessionId && "hidden")}
       >
         <TerminalPane sessionId={session.id} />
       </div>
     ))}
     {sessions.length === 0 && (
       // Existing "No active sessions" placeholder
     )}
   </div>
   ```

   This keeps all WebSocket connections alive but only shows the active terminal. Hidden terminals continue processing data in the background.

   export async function createConnection(data: Omit<Connection, 'id' | 'created_at' | 'updated_at'>): Promise<Connection> {
     const res = await fetch('/api/connections', {
       method: 'POST',
       headers: { 'Content-Type': 'application/json' },
       body: JSON.stringify(data),
     })
     if (!res.ok) throw new Error('Failed to create connection')
     return res.json()
