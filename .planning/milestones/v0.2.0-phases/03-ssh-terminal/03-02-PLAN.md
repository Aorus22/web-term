```typescript
export type SessionStatus = 'connecting' | 'connected' | 'disconnected' | 'error'

export interface SSHSession {
  id: string                     // unique session ID (matches backend session_id)
  connectionId?: string          // optional link to saved Connection
  host: string
  port: number
  username: string
  label: string                  // display label (connection.label or "user@host")
  status: SessionStatus
  error?: string                 // error message when status is 'error'
  isQuickConnect: boolean        // true if ephemeral quick-connect session
}

export interface ConnectOptions {
  // For saved connections — send connection_id, backend fetches password
  connectionId?: string
  // For quick-connect — send credentials directly
  host?: string
  port?: number
  username?: string
  password?: string
  // Initial terminal size
  rows?: number
  cols?: number
}
```

2. Create `fe/src/features/terminal/useSSHSession.ts`:

The hook encapsulates the full WebSocket lifecycle for a single SSH session. It must:
- Accept a `sessionId` parameter and provide a `connect(opts: ConnectOptions)` function
- Manage WebSocket ref internally (`useRef<WebSocket | null>(null)`)
- Integrate with @wterm/react's `useTerminal()` hook: `{ ref, write, focus }`
- Provide callbacks: `sendData(data: string)` for user keystrokes, `sendResize(cols, rows)` for terminal resize, `disconnect()` for clean close

**WebSocket lifecycle:**
- `connect()`: Create WebSocket to `ws://${window.location.hostname}:8080/ws` (same pattern as spike)
- Set `binaryType = 'arraybuffer'`
- On `open`: Send connect JSON message based on ConnectOptions:
  - If `connectionId` set: `{"type": "connect", "connection_id": "...", "rows": ..., "cols": ...}`
  - If host/user/password set: `{"type": "connect", "host": "...", "port": ..., "user": "...", "password": "...", "rows": ..., "cols": ...}`
  - Default rows=24, cols=80 if not specified
- On `message` (text): Parse JSON, handle:
  - `"connected"` → update store session status to 'connected', store session_id from response, call `focus()` after 100ms delay
  - `"error"` → update store session status to 'error' with message, close WebSocket
  - `"disconnected"` → update store session status to 'disconnected'
- On `message` (binary): Convert ArrayBuffer to Uint8Array, call `write(data)`
- On `close`: Update store session status to 'disconnected' (if not already error)
- On `error`: Update store session status to 'error' with 'WebSocket connection failed'

**Store integration:** Import `useAppStore` and call:
- `addSession()` when connect is called (status: 'connecting')
- `updateSession()` on status changes (connected, error, disconnected)

**Cleanup:** Return a cleanup function from the hook's useEffect that closes WebSocket on unmount.

**Important:** Do NOT store password in hook state. Password is sent once in the connect message and discarded. Per D-07.

Export: `useSSHSession(sessionId: string)` returning `{ ref, connect, sendData, sendResize, disconnect, write, focus }` — `ref` is the terminal ref for the `<Terminal>` component.

```typescript
// Add to AppState interface
sessions: SSHSession[]
activeSessionId: string | null
addSession: (session: SSHSession) => void
removeSession: (id: string) => void
updateSession: (id: string, updates: Partial<SSHSession>) => void
setActiveSession: (id: string | null) => void

// Add to create<AppState>((set) => ({ ... }))
sessions: [],
activeSessionId: null,
addSession: (session) => set((state) => ({ sessions: [...state.sessions, session] })),
removeSession: (id) => set((state) => ({
  sessions: state.sessions.filter((s) => s.id !== id),
  activeSessionId: state.activeSessionId === id
    ? (state.sessions[0]?.id ?? null)

updateSession: (id, updates) => set((state) => ({
  sessions: state.sessions.map((s) => s.id === id ? { ...s, ...updates } : s),

```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Terminal input → WebSocket | User keystrokes sent as binary — trusted client input |
| WebSocket → Terminal rendering | SSH server output rendered — untrusted (could be ANSI escape abuse) |
| Store → Components | Internal — trusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Information Disclosure | Password in hook state | mitigate | Password sent once in connect message, never stored in state or ref. ConnectOptions ref does not include password for saved connections (connection_id flow). |
| T-03-07 | Denial of Service | Terminal binary flood | accept | @wterm/react handles rendering performance. No rate limiting on binary frames needed for v1. |
</threat_model>
