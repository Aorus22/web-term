## Backend REST API (running on http://localhost:8080)

### GET /api/connections
Returns: `Connection[]` — all saved connections (password field omitted)
```
[{ "id": "uuid", "label": "string", "host": "string", "port": 22, "username": "string", "tags": ["string"], "created_at": "RFC3339", "updated_at": "RFC3339" }]
```

### GET /api/connections/:id
Returns: single `Connection` with decrypted password field included

### POST /api/connections
Request body: `{ "label": "req", "host": "req", "port": 22, "username": "req", "password": "opt", "tags": [] }`
Returns: created `Connection` with 201 status

### PUT /api/connections/:id
Request body: same as POST
Returns: updated `Connection`

### DELETE /api/connections/:id
Returns: 204 No Content

### GET /api/connections/export
Returns: JSON file download (`Content-Disposition: attachment`)

### POST /api/connections/import
Request body: `{ "connections": [...] }`
Returns: `{ "imported": N, "skipped": M }`
</interfaces>

   ```typescript
   const API_BASE = 'http://localhost:8080/api/connections'

   export interface Connection {
     id: string
     label: string
     host: string
     port: number
     username: string
     password?: string
     tags: string[]
     created_at: string
     updated_at: string
   }

   export const connectionsApi = {
     list: (): Promise<Connection[]> => fetch(API_BASE).then(r => r.json()),
     get: (id: string): Promise<Connection> => fetch(`${API_BASE}/${id}`).then(r => r.json()),
     create: (data: Partial<Connection>): Promise<Connection> =>
       fetch(API_BASE, { method: 'POST', headers: {'Content-Type': 'application/json'}, body: JSON.stringify(data) }).then(r => r.json()),
     update: (id: string, data: Partial<Connection>): Promise<Connection> =>
       fetch(`${API_BASE}/${id}`, { method: 'PUT', headers: {'Content-Type': 'application/json'}, body: JSON.stringify(data) }).then(r => r.json()),
     delete: (id: string): Promise<void> =>
       fetch(`${API_BASE}/${id}`, { method: 'DELETE' }).then(r => { if (!r.ok) throw new Error('Delete failed') }),
     export: (): Promise<Blob> =>
       fetch(`${API_BASE}/export`).then(r => r.blob()),
     import: (data: Connection[]): Promise<{imported: number, skipped: number}> =>
       fetch(`${API_BASE}/import`, { method: 'POST', headers: {'Content-Type': 'application/json'}, body: JSON.stringify({connections: data}) }).then(r => r.json()),
   }
   ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → REST API | Untrusted data from API responses displayed in UI |
| User input → API | Form input sent to backend (validated server-side) |
| File upload → Import | User-uploaded JSON parsed and sent to backend |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Information Disclosure | Connection list | mitigate | Password field omitted from list API responses. Form receives password only on single GET for editing. Never log passwords to console. |
| T-02-09 | Tampering | Import file | mitigate | Import data validated server-side (backend responsibility from Plan 01). Frontend shows result counts for user awareness. |
| T-02-10 | Tampering | Form input | mitigate | Client-side validation for UX (required fields, port range). Server-side validation is the authoritative check (Plan 01). |
| T-02-11 | Information Disclosure | Quick-connect input | accept | user@host:port typed in quick-connect bar is sent to backend. No XSS risk — React handles escaping. |
</threat_model>
