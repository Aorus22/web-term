## REST API Contract

### GET /api/connections
Response: `[{ "id": "uuid", "label": "string", "host": "string", "port": 22, "username": "string", "password": null, "tags": ["string"], "created_at": "RFC3339", "updated_at": "RFC3339" }]`
- Password field omitted or null in list responses (never sent in bulk)

### GET /api/connections/:id
Response: `{ "id": "uuid", "label": "string", "host": "string", "port": 22, "username": "string", "password": "decrypted-string", "tags": ["string"], ... }`
- Password included only on single-connection GET (for edit form)

### POST /api/connections
Request: `{ "label": "string", "host": "string", "port": 22, "username": "string", "password": "string", "tags": ["string"] }`
Response: created connection object with 201 status

### PUT /api/connections/:id
Request: same as POST
Response: updated connection object

### DELETE /api/connections/:id
Response: 204 No Content

### GET /api/connections/export
Response: JSON file download with all connections (passwords encrypted in export)

### POST /api/connections/import
Request: JSON body with connections array
Response: `{ "imported": N, "skipped": M }`
- Skip duplicates by host+port+username

## Connection Model (GORM)
```go
type Connection struct {
    ID        string    `json:"id" gorm:"primaryKey;type:varchar(36)"`
    Label     string    `json:"label" gorm:"not null"`
    Host      string    `json:"host" gorm:"not null"`
    Port      int       `json:"port" gorm:"default:22"`
    Username  string    `json:"username" gorm:"not null"`
    Password  string    `json:"password,omitempty"` // encrypted at rest, omitted in lists
    Tags      StringList `json:"tags" gorm:"type:text"` // JSON array stored as TEXT
    CreatedAt time.Time `json:"created_at"`
    UpdatedAt time.Time `json:"updated_at"`
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → REST API | Untrusted input via HTTP requests (no auth in v1) |
| Client → WebSocket | Untrusted WebSocket connections from browser |
| API → SQLite | Trusted internal data store |
| API → SSH target | Semi-trusted external server (user-specified host) |
| API → Encryption key | Server-side secret from env var |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | REST API handlers | mitigate | Validate all inputs: host format, port range 1-65535, required fields non-empty. Return 400 for invalid input. |
| T-02-02 | Information Disclosure | Password storage | mitigate | AES-256-GCM encryption for passwords at rest per D-19. Encryption key from env var, never logged. Passwords omitted from list responses. |
| T-02-03 | Information Disclosure | WebSocket transport | accept | TLS via reverse proxy (D-25). Document as required for non-localhost deployment. |
| T-02-04 | Spoofing | WebSocket origin | mitigate | Validate Origin header against configurable allowlist per D-28, CR-02. Default: localhost origins only. |
| T-02-05 | Elevation of Privilege | SSH proxy SSRF | mitigate | Configurable host allowlist per D-26, CR-03. Always-block cloud metadata 169.254.169.4. If allowlist empty: allow all (backward compatible). |
| T-02-06 | Tampering | JSON encoding | mitigate | Replace broken jsonEscape with json.Marshal per D-29, CR-04. Never hand-roll JSON encoding. |
| T-02-07 | Denial of Service | REST API endpoints | accept | v1 is single-user self-hosted. Rate limiting deferred to future phase. |
</threat_model>
