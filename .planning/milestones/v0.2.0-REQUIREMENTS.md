# Requirements Archive: WebTerm v0.2.0

**Milestone:** v0.2.0 MVP
**Shipped:** 2026-04-28
**Status:** All v1 requirements delivered

---

## v1 Requirements (All Complete)

### Terminal & SSH Connection

- [x] **TERM-01**: WebSocket SSH proxy — Go backend accepts WebSocket connection, dials SSH to target server, pipes bidirectional I/O ✓ Validated Phase 1, shipped Phase 3
- [x] **TERM-02**: Terminal rendering via @wterm/react — handles vim, htop, tmux, curses apps correctly ✓ Validated Phase 1, shipped Phase 3
- [x] **TERM-03**: Password-based SSH authentication with keyboard-interactive support ✓ Shipped Phase 3
- [x] **TERM-04**: Auto-resize terminal — terminal resizes when browser window/pane resizes, SSH PTY resize propagated ✓ Shipped Phase 3
- [x] **TERM-05**: Copy/paste — native browser text selection and clipboard via DOM rendering ✓ Shipped Phase 3

### Connection Management

- [x] **CONN-01**: Save SSH connections — store host, port, username, label in SQLite ✓ Shipped Phase 2
- [x] **CONN-02**: List saved connections in sidebar ✓ Shipped Phase 2
- [x] **CONN-03**: Edit saved connections ✓ Shipped Phase 2
- [x] **CONN-04**: Delete saved connections ✓ Shipped Phase 2
- [x] **CONN-05**: Quick-connect bar — type `user@host:port` and connect immediately without saving ✓ Shipped Phase 2
- [x] **CONN-06**: Connection grouping/tags — organize connections with tags/labels ✓ Shipped Phase 2
- [x] **CONN-07**: Export/import connections as JSON backup ✓ Shipped Phase 2

### Multi-Tab Sessions

- [x] **TAB-01**: Open multiple SSH sessions in tabs within single browser window ✓ Shipped Phase 4
- [x] **TAB-02**: Per-tab independent WebSocket connection and SSH session lifecycle ✓ Shipped Phase 4
- [x] **TAB-03**: Close tab cleanly disconnects SSH session (no zombie processes) ✓ Shipped Phase 4
- [x] **TAB-04**: Connection status indicator per tab — connected/connecting/disconnected ✓ Shipped Phase 4

### UI & UX

- [x] **UI-01**: Clean, modern UI using shadcn — minimalist Vercel-inspired design ✓ Shipped Phase 2
- [x] **UI-02**: Dark/light theme — system preference detection + manual toggle, synced with terminal theme ✓ Shipped Phase 4
- [x] **UI-03**: Keyboard shortcuts — Ctrl+T (new tab), Ctrl+W (close tab), Ctrl+Tab (switch tab) ✓ Shipped Phase 4
- [x] **UI-04**: Reconnection handling — auto-reconnect on WebSocket disconnect or prompt user ✓ Shipped Phase 3
